About This Project

Comparative Analysis of Circle Detection Methods for Coin Counting

Project Overview

This is an educational web application that demonstrates and compares three different computer vision algorithms for automated coin detection and counting.

Purpose: Educational demonstration of digital image processing techniques in computer vision, suitable for university-level teaching and academic publication.

Detection Algorithms

1. Morphological approach (this work)

Pipeline implemented in this project, combining classical image processing steps:

  • Blur for noise reduction (3×3 kernel)
  • HSV color space transformation and background removal (inRange: 0,70,0 to 255,255,255)
  • Otsu's adaptive thresholding for binarization
  • Morphological operations: dilation (5×5, 4 iterations) and erosion (3×3, 1 iteration)
  • Canny edge detection (thresholds: 100, 200)
  • External contour extraction (RETR_EXTERNAL, CHAIN_APPROX_SIMPLE)
  • Geometric filtering using circularity measure
Circularity = (4π × Area) / (Perimeter²)

Perfect circles: C = 1.0. Empirically determined threshold: C ≥ 0.85

2. Hough Circle Transform

Classical computer vision approach that uses parameter space voting to detect circles. Well-established method that's rotation invariant and robust to noise.

3. Blob Detection

Feature-based approach using SimpleBlobDetector with multiple criteria: area, circularity, convexity, and inertia ratio.

Performance Metrics

The application calculates and displays the following metrics:

Basic Metrics
  • Coins Detected: Number of coins found in the image
  • Processing Time: Algorithm execution time in milliseconds
Accuracy Metrics (when ground truth available)
  • Accuracy: Percentage of correctly detected coins (TP / Total Ground Truth)
  • Precision: Ratio of true positives to all detections (TP / (TP + FP))
  • Recall: Ratio of detected coins to actual coins (TP / (TP + FN))
  • F1-Score: Harmonic mean of precision and recall (2 × P × R / (P + R))
  • Mean IoU: Average Intersection over Union for matched detections
What is IoU (Intersection over Union)?

IoU measures how well a detected circle overlaps with the ground truth circle. It's calculated as the area of overlap divided by the area of union.

  • IoU = 1.0: Perfect match
  • IoU ≥ 0.5: Good detection (standard threshold)
  • IoU < 0.5: Poor detection

How to Use

  1. Upload an image containing coins (JPG or PNG format)
  2. Select which detection algorithms to run (or use all three)
  3. Click "Analyze Image" to process
  4. View side-by-side results with performance metrics
  5. Compare algorithm performance using charts

Academic Purpose

This project is developed as part of a professional paper (stručni članak) for the journal Spoznaja - Filozofski fakultet Univerziteta u Zenici.

Focus: Practical comparison of circle detection methods for educational and teaching purposes, demonstrating real-world applications of computer vision techniques.
